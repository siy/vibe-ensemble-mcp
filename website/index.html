Vibe Ensemble

Multi‑Agent Coordination MCP Server for Claude Code

What is Vibe Ensemble?

Vibe Ensemble is an MCP server that turns Claude Code into a reliable multi‑agent coordinator. You define projects and specialized worker types, create tickets with a minimal pipeline, and Vibe Ensemble handles the handoffs. Tasks progress through focused stages without context drift or focus dilution.

How it feels to use

Claude Code becomes your control tower for planning and managing several projects at once. You discuss and approve the plan as usual; once you give sign‑off, the rest runs mostly in the background. You check progress periodically, step in to resolve issues, and keep momentum without micromanaging. Claude Code shifts into a high‑level coordinator that handles the routine. Because it’s not writing code or diving into every detail itself, its focus stays sharp and context remains clean for much longer.

Key Capabilities

Focused, staged execution

Specialized workers (planning, design, implementation, testing, review) process tickets in well‑defined stages.

Advanced planning and sizing

Built‑in methodology to size tasks for speed and reliability; planners output complete pipelines and JSON coordination.

Automatic handoffs

Finishing a stage updates the ticket and queues the next worker. Recovery on restart is built‑in.

Multiple projects

Coordinate several projects simultaneously without losing context or mixing priorities.

Real‑time monitoring

HTTP MCP + SSE with events and status updates. See what’s moving, act where needed.

Safer defaults

Permission modes for workers (inherit, file, bypass) so you can choose the right level of access.

How it works

Coordinator

Claude Code acts as a coordinator. It manages projects, defines worker types, creates tickets, reviews events, and resumes stalled work. It does not implement tasks itself.

Planner

Transforms a goal into a staged plan using the sizing methodology. Validates worker types and outputs a strict JSON block with outcome, target stage, pipeline update, comment, and reason.

Server

The Rust MCP server manages SQLite state, queues, process execution, and MCP/SSE integrations. Stage completion automatically enqueues the next stage.

Quick Install

Install the server, configure Claude Code, and get moving in minutes.

curl -fsSL https://get.vibeensemble.dev/install.sh | sh
iwr -useb https://get.vibeensemble.dev/install.ps1 | iex

Usage

Configure Claude Code integration and start the server.

vibe-ensemble-mcp --configure-claude-code --host 127.0.0.1 --port 3000
vibe-ensemble-mcp --port 3000
vibe-ensemble

Planner & Coordinator Details

Planner outputs

Complete pipeline, worker type checks, JSON outcome including target stage, optional pipeline update, and clear handoff comment and rationale.

Coordinator responsibilities

Create projects and worker types, create tickets starting at planning, monitor events, handle escalations, and use resume_ticket_processing to keep work flowing.

Ticket lifecycle

Create → Plan → Stage N → Next stage (auto) → Review → Close. All state is persisted with automatic recovery for open, unclaimed tickets.

Security & Status

  • Worker permission modes: inherit (default), file (settings from .vibe-ensemble-mcp/worker-permissions.json), or bypass (not recommended).
  • SSE and MCP integrations are evolving.
  • Interfaces may change between early releases.
  • Use in controlled environments and review the repository README for the latest warnings.
